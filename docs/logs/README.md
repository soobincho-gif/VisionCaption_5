# Project Logs

This folder is the report-oriented project log archive.

Purpose:
- keep milestone plans in one place,
- record what was solved,
- record what changed,
- preserve implementation rationale for later reporting,
- make future handoff and reconstruction easier.

How to use:
- create one file per milestone or focused work session,
- keep the structure consistent,
- link relevant code, docs, and outputs,
- avoid putting secrets in any log file.

Suggested log structure:
1. Objective
2. Why this task mattered
3. Plan
4. What was implemented
5. What changed
6. Validation
7. Remaining issues
8. Next task
